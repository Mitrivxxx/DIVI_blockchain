SPRINT 4 — BACKEND (INTEGRACJA)

Cel sprintu:
Stworzenie jednego, spójnego API backendowego, które integruje przechowywanie
dokumentów w IPFS oraz rejestrację ich w blockchainie.
Użytkownik komunikuje się tylko z backendem, który zarządza całą logiką i
bezpieczeństwem.

Główne zadania:

1. Stworzenie endpointów API:
	- **publishDocument** — przyjmuje dokument, zapisuje go w IPFS, a następnie rejestruje hash w smart kontrakcie na blockchainie.
	- **verifyDocument** — sprawdza, czy dany hash dokumentu istnieje w smart kontrakcie (weryfikacja autentyczności).

2. Integracja z Web3:
	- Wykorzystanie bibliotek ethers.js do komunikacji z blockchainem Ethereum.

3. Obsługa błędów:
	- Brak CID z IPFS (np. błąd zapisu pliku)
	- Błąd zapisu do blockchaina
	- Brak uprawnień użytkownika do wykonania operacji
	- Odpowiednie komunikaty zwrotne w API

Efekty sprintu (Definition of Done):

✔ Jedno API — jeden punkt prawdy dla dokumentów (wszystko przez backend)
✔ Backend poprawnie komunikuje się zarówno z IPFS, jak i z blockchainem
✔ Przypadki błędów są obsłużone i odpowiednio raportowane
✔ Kod jest przetestowany (testy integracyjne)

-----------------------------------
1. Frontend → upload pliku
2. Backend → generuje hash + wysyła do IPFS → zwraca CID + hash
3. Frontend → wywołuje smart kontrakt przez ethers.js
4. MetaMask → podpis
5. Transakcja → blockchain
